Streaming API Test SaaS Platform

Goals

  1. Evaluate ServerSideEvents as a steaming mechanism
  2. Evaluate WebSockets as a steaming mechanism
  3. Investigate the feasibility of attaching streaming interfaces to dynamic resources (such as the SaaS platform guest systems instances)


  • REST and Streaming interfaces to shared dynamic resources
    • List Methods
    • Expand Methods

    • We prototype creating dynamic resources with the PUT /log API.
    • The same resources can then be either queried with the usual REST GET /log/{id} call.
    • Or alternatively through the streaming GET /log/sse/{id} interface
